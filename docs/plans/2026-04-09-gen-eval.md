# Gen Eval Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `gen-eval` subcommand that lets one code-agent task repair an operator in place when needed, generate test and benchmark harnesses, and validate both artifacts locally or through remote-aware execution.

**Architecture:** Keep `gen-eval` in the same thin-wrapper style as the other agent-backed commands. Add one new command kind, one new handler, one dedicated workflow skill, and a small extension to skill staging so this command can request a restricted staged-skill set while the rest of the CLI keeps its current behavior.

**Tech Stack:** Python `argparse`, `dataclasses`, existing agent runner infrastructure, Markdown skills, Python `unittest`
